Last updated: April 2026

STEP 1 — SUPABASE DATABASE SETUP
──────────────────────────────────
1. Go to supabase.com → your project
2. Click "SQL Editor" in left sidebar
3. Click "New Query"
4. Open supabase_migration_v2.sql and paste entire contents
5. Click "Run"
6. You should see a list of 9 tables created

STEP 2 — GET YOUR SUPABASE CREDENTIALS
────────────────────────────────────────
In Supabase dashboard:
  → Project Settings → API
  → Copy "Project URL"       (SUPABASE_URL)
  → Copy "anon public" key   (SUPABASE_KEY)

STEP 3 — RAILWAY BACKEND DEPLOYMENT
─────────────────────────────────────
1. Go to railway.app
2. Login with your GitHub account (philanthropistai)
3. New Project → Deploy from GitHub repo
4. Select: ilikeikm7-jurivon-backend
5. Wait for first deploy to finish (it will fail — no env vars yet)
6. Click your service → Variables tab → Add these:

   OPENAI_API_KEY    = sk-your-openai-key-here
   SUPABASE_URL      = https://lznzlabvnhugirfogaqq.supabase.co
   SUPABASE_KEY      = your-supabase-anon-key
   ENVIRONMENT       = production
   FRONTEND_URL      = https://jurivon-frontend.vercel.app

7. Railway will auto-redeploy after saving variables
8. Copy your Railway URL (looks like: https://web-production-xxxxx.up.railway.app)
9. Test: open https://YOUR-RAILWAY-URL/health in browser
   Should return: {"status":"ok","version":"2.0.0","database":"connected"}

STEP 4 — VERCEL FRONTEND DEPLOYMENT
─────────────────────────────────────
1. Go to vercel.com → New Project
2. Import your GitHub repo (or drag and drop index.html)
3. No build settings needed — it is a static HTML file
4. Deploy
5. Copy your Vercel URL

STEP 5 — CONNECT FRONTEND TO BACKEND
──────────────────────────────────────
1. Open your Vercel URL in browser
2. Click Settings (⚙️) in the sidebar
3. Paste your Railway URL in "Backend URL" field
4. Enter your firm name and firm ID
5. Click "Test Connection" — should show "Connected ✓"
6. Click "Save Settings"

STEP 6 — LOCAL DEVELOPMENT SETUP (Windows)
─────────────────────────────────────────────
1. Open PowerShell
2. Navigate to your backend folder:
   cd C:\Users\Marwat\Desktop\jurivon-backend

3. Create virtual environment:
   python -m venv venv
   venv\Scripts\activate

4. Install dependencies:
   pip install -r requirements.txt

5. Create .env file in the backend folder with this content:
   OPENAI_API_KEY=sk-your-key
   SUPABASE_URL=https://lznzlabvnhugirfogaqq.supabase.co
   SUPABASE_KEY=your-key
   ENVIRONMENT=development
   FRONTEND_URL=http://localhost:5500

6. Run locally:
   uvicorn main:app --reload --port 8000

7. Test:
   Open http://localhost:8000/health

STEP 7 — PUSH UPDATES TO RAILWAY
──────────────────────────────────
After making any changes:

   git add .
   git commit -m "describe what you changed"
   git push origin main

Railway auto-deploys from every push.

STEP 8 — PAKISTAN LAW INGESTION
─────────────────────────────────
Run this weekly on your local PC:

   python ingest_pakistan_law.py

Before running, add PDF URLs to the DOCUMENTS_TO_INGEST list
in ingest_pakistan_law.py. Get PDFs from:
  - supremecourt.gov.pk/judgments/
  - pakistancode.gov.pk/
  - federalshariatcourt.gov.pk/
  - punjablaws.gov.pk/

TROUBLESHOOTING
────────────────
Problem: "Connection error" on frontend
Fix: Check Backend URL in Settings. Test with /health endpoint.

Problem: Railway app sleeping / not responding
Fix: Upgrade to Railway Hobby plan ($5/month). Free tier sleeps after 30min.

Problem: Conflict check returns no database results
Fix: Add matters in Matter Database panel first.

Problem: Pakistan law research shows "AI knowledge (verify)"
Fix: Run ingest_pakistan_law.py with real PDF URLs first.

Problem: OdV feature not visible
Fix: Settings → check "Show OdV Art. 231 (Italian firms only)" → Save.

Problem: Streaming draft stops mid-way
Fix: Railway timeout — check Railway logs. Increase timeout in uvicorn.
